The Gamesim Class Specification

This class controls the simulation of an NFL game

Ryan Abdella. XML version of specification written by Ryan Abdella, Wednesday, November 30, 2011




CLASS gamesim

No superclass.

State variables:

float gametime
The time left in the current quarter.

team * poss
Points to the team that is on offense.

team * notposs
Points to the team that is on defense.

team * initposs
Points to the team that received to start the game.

int togo
Yardage until goalline.

int down
The current down.

int tofirst
Yardage until a first down.

int qtr
The current quarter.

int position
The postion on the field from 0 to 100.

string event1
Most recent play.

string event2
Second most recent play.

string event3
Third most recent play.

string event4
Fourth most recent play.

string event5
Fifth most recent play.

string event
Temp string for constructing output.

team home
The home team.

away away
The away team.

ptype playtype
The current play type.

int yardage
The length of the current play.

bool fumble
Used to determine if a fumble occured.

bool interception
Used to determine if an interception occured.

bool sack
Used to determine if a sack occured.

bool complete
Used to determine if a pass was complete.

bool fg
Used to determine if a field goal was good.

bool touchback
Used to determine if a punt of kickoff was a touchback.

bool xp
Used to determine if an extra point was good.

bool td
Used to determine if a touchdown occurred.

int returnlength
The length of the punt return.

int puntlength
The length of the punt.

Constructors:

gamesim
Default Constructor.

No arguments.

State change:
This instance of gamesim is initialized by assigning everything the regular constructor does except for the teams.

gamesim
Regular Constructor.

Arguments:

team t1
The home team.

team t2
The away team.

State change:
This instance of gamesim sets all bools to false, all strings to "", the playtype to kickoff, yardage to 0, gametime to 900, qtr to 1, down to 1, tofirst to 10 and togo to 100.

Destructor:
~gamesim

State change:

Methods:

void simdown
This method will simulate one play.

No arguments.

State change:
Updates the state variables to reflect the outcome of the play.

Return value:
None.

void gameTime
This method will determine the amount of time a play took.

No arguments.

State change:
Subtracts the time for the given play from the time left in the quarter. If a quarter ends, changes the necessary state variables for the next quarter.

Return value:
None.

void updateYardage
This method updates the various state variables that have to do with field position.

No arguments.

State change:
State variables that have to do with field position are updated after a play is run.

Return value:
None.

ptype cpuPlayType
Determines what type of play the cpu would run in a given situation.

No arguments.

State change:
None.

Return value:
New playtype.

void changePoss
Changes the pointers poss and notposs to the other team.

No arguments.

State change:
Teams that poss and notposs point to are switched.

Return value:
None.

void kickoff
Simulates a kickoff.

No arguments.

State change:
Changes yardage to reflect the result of the kickoff.

Return value:
None.

void runPlay
Simulates a rushing play.

No arguments.

State change:
Changes yardage and fumble to reflect the result of the play.

Return value:
None.

void passPlay
Simulates a passing play.

No arguments.

State change:
Changes yardage, fumble, sack, and interception to reflect the result of the play.

Return value:
None.

void punt
Simulates a punt play.

No arguments.

State change:
Changes yardage, puntlength, and returnlength to reflect the result of the play.

Return value:
None.

void fieldgoal
Simulates a field goal.

No arguments.

State change:
Changes yardage and fg to reflect hte result of the play.

Return value:
None.

void extrapoint
Simulates an extra point.

No arguments.

State change:
Changes xp and the team with possessions score to reflect the result of the play.

Return value:
None.

void setTeams
Sets the pointers to the teams used for the simulation.

Arguments:

team * t1
A pointer to a team.

team * t2
A pointer to a team.

State change:
Changes where the pointers home and away point to.

Return value:
None.

void updateOutput
Updates the strings that detail the previous plays.

No arguments.

State change:
Changes the event strings.

Return value:
None.

float getcomppercent
Averages the quarterback, receiver, and defense to get a real completion percent.

Arguments:

player * p
A pointer to the receiver.

State change:
None.

Return value:
A decimal representing a completion percentage.

float getypcatch
Average the quarterback, receiver, and defense to get an accurate yards per catch.

Arguments:

player * p
A pointer to the receiver.

State change:
None.

Return value:
A number representing an average yards per catch.

float getinterrate
Averages the quarterback and defense to get a real interception rate.

No arguments.

State change:
None.

Return value:
A decimal representing an interception rate.

float getsackrate
Averages the quarterback and defense to get a real sack rate.

No arguments.

State change:
None.

Return value:
A decimal representing a sack rate.

float getypr
Averages the runningback and defense to get an accurate yards per rush.

Arguments:

player * p
A pointer to the runningback.

State change:
None.

Return value:
A number representing an average yards per rush.

float getfumbrate
Averages the player and defense to get a real fumble rate.

Arguments:

player * p
A pointer to the player.

State change:
None.

Return value:
A decimal representing a fumble rate.

void start
Starts a game simulation.

No arguments.

State change:
Changes the state variables associated with yardage and possession to values used in the beginning of the game.

Return value:
None.
